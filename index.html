Frequently Asked Questions

1. What is the difference between var, let, and const?
In JavaScript, var, let, and const are used to declare variables, but they work differently. var is old and can be used nywhere inside a function, but it can cause problems because it gets hoisted (moved to the top) and allows redeclaring the same variable. let is better because it only works inside the block {} where it is declared and doesn't let you use it before it's assigned. const is like let, but once you set a value, you can’t change it. However, if it's an object or array, you can still modify its contents. It's best to use let for changing values and const for values that stay the same.
2. What is the difference between map(), forEach(), and filter()?
In JavaScript, map(), forEach(), and filter() are used to work with arrays, but they have different purposes. map() is used when you want to create a new array by modifying each element of the original array without changing it. It returns a new array with the transformed values. forEach() is different because it does not create a new array; instead, it just loops through each element and performs an action, like printing values or updating something, but it doesn’t return anything. filter() is useful when you only want certain elements from an array based on a condition. It checks each element and creates a new array with only the elements that meet the condition. So, map() is great for changing values, forEach() is good for running a function on each item without making a new array, and filter() helps in selecting specific values.
3. Explain arrow functions and how they are different from regular functions?
In JavaScript, arrow functions are a shorter way to write functions. They use => instead of the function keyword, making the code cleaner and easier to read. One big difference between arrow functions and regular functions is how they handle this. In regular functions, this changes depending on how the function is called, which can sometimes cause confusion. But in arrow functions, this always refers to the surrounding scope, meaning it doesn’t change. Another difference is that arrow functions cannot be used as constructors, so you can’t create objects with them using new. Also, they don’t have their own arguments object like regular functions do. Arrow functions are great for short and simple functions, but for complex functions that need this or arguments, regular functions are still useful.
4. How JavaScript Promises work?
In JavaScript, Promises are used to handle asynchronous operations, like fetching data from a server. A Promise represents a task that will complete in the future and can either resolve (success) or reject (failure). When a Promise is created, it starts running immediately and eventually returns a result. We use .then() to handle success and .catch() to handle errors. Promises help avoid callback hell, making the code cleaner and easier to read. They also support chaining, meaning we can run multiple asynchronous tasks one after another. If something goes wrong in any step, the .catch() method will catch the error. Promises are very useful when working with APIs, databases, or any task that takes time to complete.
5. How closures work in JavaScript?
In JavaScript, a closure happens when a function remembers and can access variables from its outer function even after the outer function has finished running. This works because inner functions keep a reference to the variables of their parent function. Closures are useful for creating private variables, because the outer function’s variables cannot be accessed directly from outside. They also help in scenarios like data hiding, maintaining state, and creating functions with specific behaviors. Since closures keep variables alive in memory, they should be used carefully to avoid unnecessary memory usage. They are commonly used in event handlers, timers, and callbacks to maintain values over time.
